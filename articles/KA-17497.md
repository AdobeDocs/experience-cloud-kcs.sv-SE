# Ta reda på vilka förfrågningar AEM Dispatcher Cache

## Beskrivning {#description}

<br>Problem<br><br>
Måste avgöra varför HTTP-begäran/svar inte cachas av AEM Dispatcher
<br><br>Miljö<br><br>
AEM Dispatcher 4.3.3


## Upplösning {#resolution}


Dokumentationen innehåller de viktigaste scenarierna som påverkar cachemöjligheterna för begäranden och svar som går igenom AEM Dispatcher-modulen. Alla cacheregler täcks inte.

<b>Regler för HTTP-begäran och -cachelagring</b>

För att en HTTP-begäran ska kunna nås av avsändaren måste den följa följande regler:

1. URL är absolut (fungerar inte med . eller ..) har ett filtillägg och filnamnet i URL:en fungerar som en fil i filsystemet.
   <b>Relaterade loggmeddelanden:</b>


   ```
   URI not canonical: %s.    
   Unable to map URI to file: %s.    
   cache file path too long    
   temporary file path too long    
   request URL has no extension
   ```



   <b>Exempel:</b>


   | `/content/test` | inte cachelagrad |
   | --- | --- |
   | `/content/test.html` | cachelagrad |


2. Det har inga snedstreck efter det första filtillägget.<br>

   <b>Relaterade loggmeddelanden:</b>


   ```
   request URL has trailing slash
   ```



   <b>Exempel:</b>


   | `/content/test.html/` | inte cachelagrad |
   | --- | --- |
   | `/content/test.html/foo.jpg` | inte cachelagrad |
   | `/content/test.html` | cachelagrad |


3. Den använder HTTP-GET eller HEAD.<br>

   <b>Relaterade loggmeddelanden:</b>


   ```
   request method is neither GET nor HEAD
   ```



   <b>Exempel:</b>


   | `HEAD /content/test.html HTTP/1.1` | cachelagrad |
   | --- | --- |
   | `GET /content/dam/test.jpg HTTP/1.1` | inte cachelagrad |
   | `POST /content/test.html HTTP/1.1` | inte cachelagrad |


4. Begäran nekas av /CACHE = /REGLER som definieras i dispatchern. EVENTUELL GRUPPKONFIGURATION.
   <b>Relaterade loggmeddelanden:</b>


   ```
   URI not  in cache rules: %s request URL not in cache rules
   ```



   <b>Exempel:</b>


   | <br><br>    ```<br>    Farm has /cache = /rules and a request is received for /content/test.html<br>        /cache<br>        {<br>         ...<br>         /rules<br>         {<br>          /0001 {<br>          type "deny"<br>          glob "*"<br>          }<br>         }<br>        }<br>    ```<br><br> | inte cachelagrad |
   | --- | --- |
   | <br><br>    ```<br>    Farm has /cache = /rules and a request is received for /content/test.html<br>        /cache<br>        {<br>         ...<br>         /rules<br>         {<br>          /0001 {<br>          type "allow"<br>          glob "*"<br>          }<br>         }<br>        }<br>    ```<br><br> | cachelagrad |


5. HTTP-begäran innehåller inte någon auktoriseringshuvud eller så innehåller den rubriken, men /ALLOWAUTHZED är inställd på 1 i FARM CONFIGURATION.
   <b>Relaterade loggmeddelanden:</b>


   ```
   request contains authorization
   ```



   <b>Exempel:</b>


   | HTTP-begäran:<br><br>    ```<br>    GET /content/test.html HTTP/1.1    Authorization: Basic YWxhZGRpbjpvcGVuc2VzYW1l<br>        dispatcher.any farm /cache = /allowAuthorized configuration:<br>        /cache {<br>           /allowAuthorized “0”    }<br>    ```<br><br> | inte cachelagrad |
   | --- | --- |
   | HTTP-begäran:<br><br>    ```<br>    GET /content/test.html HTTP/1.1    Authorization: Basic YWxhZGRpbjpvcGVuc2VzYW1l<br>        dispatcher.any farm /cache = /allowAuthorized configuration:<br>        /cache {<br>           /allowAuthorized “1”    }<br>    ```<br><br> | cachelagrad |


6. HTTP-begäran innehåller LOGIN-TOKEN eller Authorization COOKIE i cookie-huvudet eller innehåller någon (eller båda) av dessa cookies och /ALLOWAUTHZED är inställd på 1 i FARM CONFIGURATION.
   <b>Relaterade loggmeddelanden:</b>


   ```
   request contains authorization
   ```



   <b>Exempel:</b>


   | HTTP-begäran:<br><br>    ```<br>    GET /content/test.html HTTP/1.1    Cookie: login-token=…<br>        dispatcher.any farm /cache = /allowAuthorized configuration:<br>        /cache {<br>           /allowAuthorized “0”    }<br>    ```<br><br> | inte cachelagrad |
   | --- | --- |
   | HTTP-begäran:<br><br>    ```<br>    GET /content/test.html HTTP/1.1    Cookie: authorization=YWxhZGRpbjpvcGVuc2VzYW1l<br>        dispatcher.any farm /cache = /allowAuthorized configuration:<br>        /cache {<br>           /allowAuthorized “0”    }<br>    ```<br><br> | inte cachelagrad |
   | HTTP-begäran:<br><br>    ```<br>    GET /content/test.html HTTP/1.1    Cookie: login-token=…<br>        dispatcher.any farm /cache = /allowAuthorized configuration:<br>        /cache {<br>           /allowAuthorized “1”    }<br>    ```<br><br> | cachelagrad |


7. Det finns en QUERYSTRING-parameter i URL:en och parametern kan ignoreras via FARM:S /IGNOREURLPARAMS CONFIGURATION.
   <b>Relaterade loggmeddelanden:</b>


   ```
   request contains a query string
   ```



   <b>Exempel:</b>


   | HTTP-begäran:<br><br>    ```<br>    GET /content/test.html?test=1 HTTP/1.1    Farm /cache configuration:<br>        /ignoreUrlParams {     /0001 { /type “allow” /glob “*” }    }<br>    ```<br><br> | cachelagrad |
   | --- | --- |
   | HTTP-begäran:<br><br>    ```<br>    GET /content/test.html?test=1 HTTP/1.1    Farm /cache configuration:<br>        /ignoreUrlParams {     /0001 { /type “deny” /glob “*” } <br>           /0001 { /type “allow” /glob “test” }    }<br>    ```<br><br> | cachelagrad |
   | HTTP-begäran:<br><br>    ```<br>    GET /content/test.html?test=1 HTTP/1.1    dispatcher.any farm /cache = /allowAuthorized configuration:<br>        /ignoreUrlParams {<br>         /0001 { /type “deny” /glob “*” } <br>           /0001 { /type “allow” /glob “q” }    }<br>    ```<br><br> | inte cachelagrad |



<b>HTTP-svar och cachningsregler:</b>

HTTP-svaret som returneras från AEM kan nås om följande villkor uppfylls:

1. Avsändaren kan skicka och ta emot 200 OK-svar från en av de definierade RENDERS-alternativen.

   <b>Obs!</b> Om nej [/timeout](https://experienceleague.adobe.com/docs/experience-manager-dispatcher/using/configuring/dispatcher-configuration.html?lang=en) är inställt på /renders eller är inställt på 0, så väntar det för alltid på en anslutning till AEM även om instansen är nere.
   <b>Relaterade loggmeddelanden:</b>


   ```
   Unable to send request to remote server. Unable to receive response from remote server. Remote server returned: %s No backend available.
   ```



   <b>Exempel:</b>


   | HTTP-begäran:<br><br>    200 OK | cachelagrad |
   | --- | --- |
   | HTTP-begäran:<br><br>    404 Sidan hittades inte | inte cachelagrad |


2. Inga av dessa svarshuvuden finns med i svaret:

   - `Dispatcher: no-cache`
   - `Cache-control: no-cache`
   - `Pragma: no-cache`

   <b>Relaterade loggmeddelanden:</b>


   ```
   Backend forbids caching: %s, sent: %s" response contains no_cache
   ```



   <b>Exempel:</b>


   | HTTP-svar:<br><br>    200 OK<br><br>    Dispatcher: no-cache | inte cachelagrad |
   | --- | --- |
   | HTTP-svar:<br>    <br><br>    200 OK<br><br>    Cache-control: no-cache | inte cachelagrad |
   | HTTP-svar:<br><br>    200 OK<br><br>    Pragma: no-cache | inte cachelagrad |
   | HTTP-svar:<br><br>    200 OK<br> | cachelagrad |


3. Värdet för innehållslängd är större än ZERO BYTES.
   <b>Relaterade loggmeddelanden:</b>


   ```
   response content length is zero
   ```



   <b>Exempel:</b>


   | HTTP-svar:<br><br>    200 OK<br><br>    Content-Length: 0 | inte cachelagrad |
   | --- | --- |
   | HTTP-svar:<br><br>    200 OK<br><br>    Content-Length: 3120 | cachelagrad |
