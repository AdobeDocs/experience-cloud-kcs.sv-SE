# Optimera Oak Lucene-index för att minska indexeringstiden

## Beskrivning


Optimera AEM Oak-indexering för att minska omindexeringstiden.
<br><br>Miljö<br><br>
AEM 6.0 + ek 1.0.16 eller senare AEM 6.1 och AEM 6.2


## Upplösning


Du kan optimera indexen för Oak Lucene genom att finjustera eken: QueryIndexDefinition med egenskaper som &quot;includedPaths&quot;, &quot;excludePaths&quot;, &quot;name&quot; och &quot;indexPath&quot;.

- Egenskaperna &quot;includedPaths&quot; och &quot;excludePaths&quot; begränsar vilka sökvägar indexet går igenom vid omindexering och vilka sökvägar det lyssnar efter indexuppdateringar.
- Egenskaperna &quot;name&quot; och &quot;indexPath&quot; gör indexloggarna lättare att förstå eftersom de taggar loggmeddelandena med sökvägen eller namnet på indexet.  Exempel:



| `30.01.2017 14:07:12.934 *DEBUG* pool-8-thread-2 org.apache.jackrabbit.oak.plugins.index.lucene.LuceneIndexEditor damassetlucene(` `/oak` `:index` `/damAssetLucene` `) = Indexed 9000 nodes...` |
| --- |


<b>Steg för att optimera index</b>:

1. Gå till http://*aem-host:port*/crx/de/index.jsp och logga in som administratör
2. Bläddra till /oak:index
3. Lägg till egenskaperna som är markerade under 1 i motsvarande index.  The <b>includedPaths</b> och <b>excludePaths</b> ska vara en sträng med flera värden ( String )
4. Klicka på &quot;[!UICONTROL Save All]&quot; i CRXDe
